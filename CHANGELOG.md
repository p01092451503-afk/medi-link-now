# Find-ER (파인더) 작업 이력서

## 📋 개요

본 문서는 Find-ER 프로젝트의 UI/UX 및 기능 변경 작업 내역을 정리한 문서입니다.

---

## 🏥 병원 상세 바텀시트 — 액션 버튼 노출 로직 개선

### 배경

병원 마커 클릭 시 나타나는 상세 바텀시트(HospitalBottomSheet)의 액션 버튼이 모든 사용자 모드에서 동일하게 표시되어, 사용자 상황에 맞지 않는 버튼이 노출되는 문제가 있었습니다.

### 변경 사항

#### 1. 119 전화하기 버튼 — 보호자 비응급 모드에서 제거

- **변경 전**: 모든 모드에서 119 전화하기 버튼이 표시됨
- **변경 후**: 보호자/환자의 비응급(이송) 모드에서는 119 버튼을 숨김
- **이유**: 비응급 이송 상황에서는 119 호출이 불필요하며, 사설구급차 부르기 버튼이 더 적합

#### 2. 119 전화하기 버튼 — 전문가 모드 전용 표시

- **변경 전**: 보호자 모드에서도 119 버튼이 무조건 노출
- **변경 후**: 119 구급대원 모드, 사설구급차 기사 모드에서만 119 버튼 표시
- **이유**: 전문 구급 인력이 병원과 직접 소통할 때 필요한 기능

#### 3. 보호자 119 응급 모드 — 119 버튼 복원

- **변경 전**: 보호자 모드에서 119 버튼이 완전히 숨겨짐
- **변경 후**: 보호자/환자의 "119 응급" 모드에서는 병원명 바로 아래에 119 전화하기 버튼을 표시
- **이유**: 응급 상황에서는 119 호출이 최우선이므로 최상단 배치로 접근성 확보

### 모드별 버튼 노출 매트릭스

| 사용자 모드 | 119 전화하기 | 사설구급차 부르기 | 디지털 이송 요청 |
|:---|:---:|:---:|:---:|
| **보호자/환자 — 119 응급** | ✅ 병원명 아래 최상단 | ✅ 하단 보조 | ❌ |
| **보호자/환자 — 비응급(이송)** | ❌ | ✅ | ✅ |
| **119 구급대원** | ✅ | ❌ | ✅ |
| **사설구급차 기사** | ✅ | ❌ | ✅ |

### 핵심 구현 코드

```tsx
// 119 전화하기 버튼 조건부 렌더링
{(isParamedicMode || isDriverMode || !isTransferMode) && (
  <button onClick={() => { window.location.href = "tel:119"; }}
    className="w-full mb-4 py-4 rounded-2xl bg-destructive text-destructive-foreground ...">
    119 전화하기
  </button>
)}
```

## 🎨 UI/UX 디자인 원칙

### 119 전화하기 버튼

- **색상**: `bg-destructive` (빨간색 계열) — 긴급 상황 시각적 강조
- **위치**: 병원명 직후 최상단 배치 (응급 모드)
- **크기**: `w-full py-4` — 터치 영역 확보 (모바일 우선)
- **아이콘**: `Phone` (lucide-react) + 텍스트 조합

### 사설구급차 부르기 버튼

- **역할**: 비응급 이송 시 메인 CTA
- **배치**: 응급 모드에서는 하단 보조, 비응급 모드에서는 메인
